## Tracing

It's really easy to add tracing to your Vercel AI SDK application with Montelo.

Simply start a new trace at the top of your route handler:

```typescript api/chat/route.ts
import { Montelo } from "montelo";

const montelo = new Montelo();

export async function POST(req: Request) {
  const { messages } = await req.json();

  const trace = montelo.trace({ name: "Chat" });

  const res1 = await trace.openai.chat.completions.create({
    model: "gpt-4",
    messages,
    stream: true,
  });

  const res2 = await trace.anthropic.messages.create({
    model: "claude-3-sonnet-20240229",
    messages,
    max_tokens: 100,
    stream: true,
  });
}
```

Both `openai` and `anthropic` logs will be grouped together under the same trace, making it easy to see the full picture
of what's happening in your application.

The `trace` object is used to group together all the logs that are part of the same trace. It exposes the same methods as
`new Montelo()`, so you can use it to make API calls.

Check out the documentation on [Tracing](docs/traces/traces) for more information.

### Trace ID

The trace ID is a unique identifier for the trace. It is used to group together all the logs that are part of the same trace.

If you'd like to use a custom trace ID, like a chat or session ID, you can pass in `id` to the `trace` method:

```typescript
const trace = montelo.trace({ name: "Chat", id: "chat-123" });
```
